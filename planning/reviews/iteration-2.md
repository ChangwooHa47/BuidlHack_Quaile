5. **contract-03a `contribute()` flow unimplementable** — **MOSTLY CLOSED.** The rewrite (lines 82-246) is now in terms of real Promise API: entry method takes `#[payable]`, writes Pending record + reserves nonce synchronously, launches `ext_policy_registry::get_policy` → `on_get_policy` → `ext_verifier::is_eligible` → `on_is_eligible` chain, with an explicit rollback helper. This is exactly the trap iter-1 flagged. (b) `on_get_policy`/`on_is_eligible` rollback says "return `Promise::new(investor).transfer(staged_amount)`" but the callbacks' return types are `Promise` and `PromiseOrValue<bool>` — the staged amount is in the `Contribution` record, not captured in the callback args; the pseudocode reads it via `rollback_contribution(...).amount.0` but `on_is_eligible`'s signature (line 113-120) does NOT take `nonce: [u8;32]` even though the rollback helper requires it, while the `#[ext_contract]` trait at lines 270-279 DOES include `nonce: [u8;32]`. Signatures don't match. (c) Gas annotations say 30 TGas views + 60 TGas callback but no backpressure budget for `ext_verifier::is_eligible` which takes a large `AttestationBundle` (can be >8KB Borsh).

6. **Cross-contract `is_eligible(bundle)` size/gas** — **PARTIAL.** contract-03a line 96 now annotates a total budget of ~120 TGas. ## Overall

**Not yet "개발만 하면 될 퀄리티" — but much closer.** Iteration 2 correctly identified the hardest architectural problems (Promise chain rewrite, outcome+flags model, canonical message, settle/mark_closed split, nonce key formula) and either solved or scoped them. ERD v2 is a real improvement and the missing tasks were added. The remaining gap is entirely **mechanical propagation failure**: decisions were made in one file but not copy-edited into the other files that a Dev Agent will actually read at implementation time. The single most damaging case is that `contract-03a` simultaneously declares the OLD `ContributionStatus` enum as its state type AND writes the NEW `outcome/claim_done/refund_done` fields in its own pseudocode — a file that cannot compile against itself. A dev would reach this wall in the first 10 minutes. Iteration 3 does not need new thinking; it needs a careful text sweep of 8-10 specific locations and one honest rewrite of `contract-03c`. After that, blockers are zero and the plan is implementable.
